/**
 * ✓ AUTHENTICATION: User verified at proxy.ts
 *   → extractUserContextFromHeader() reads x-user-id header
 *   → Header set only by proxy.ts after JWT verification
 *   → No JWT decoding needed in handler
 *   → Cannot be bypassed without valid token
 *
 * ✓ AUTHORIZATION: RPC enforces privacy
 *   → get_nearby_users() respects friend relationships
 *   → Respects privacy settings (is_ghost_mode, is_public)
 *   → User can only see authorized users
 *
 * ✓ INPUT VALIDATION: Strict coordinate checking
 *   → lat must be -90 to 90
 *   → lng must be -180 to 180
 *   → Missing or non-numeric values rejected
 *   → Prevents invalid PostGIS queries
 *
 * ✓ HEADER SPOOFING PREVENTION
 *   → Client cannot inject x-user-id (it's set by proxy.ts)
 *   → Even if client sends custom x-user-id, proxy sanitizes it
 *   → Only proxy-verified value is trusted
 *
 * ✓ CORS SECURITY
 *   → buildCorsHeaders() called automatically by jsonResponse()
 *   → Whitelisted origins only (no wildcard for authenticated requests)
 *   → Credentials support for Flutter mobile app
 *
 * ✓ CACHE CONTROL
 *   → Auth failures set Cache-Control: no-store
 *   → Prevents caching of auth errors
 *   → Success responses can be cached (per buildCorsHeaders)
 *
 * ✓ ERROR SAFETY
 *   → Generic error messages (no stack traces)
 *   → Detailed logging for debugging (server-side only)
 *   → User context in all error logs (audit trail)
 */
